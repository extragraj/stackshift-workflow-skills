# v0.2.0 — Seeding Strategy Support

## What changed

Introduces the seeding strategy system — a mechanism for activating reusable content scaffold strategies that pre-fill `initialValue/` in Sanity Studio sections. The first strategy ships with this release: **Initial-Value Seeding**, which extracts text from HTML mockups and hardcoded components into realistic placeholder copy.

---

## Architecture

Seeds use a dual-location design:

- **Canonical content** lives in `stackshift-core/seeds/<name>.md` — always on disk because `stackshift-core` is always installed.
- **Discoverable stub** lives in `skills/stackshift-seed-<name>/SKILL.md` — a top-level skill folder that `npx skills add` can find, containing only frontmatter and a reference pointer to the canonical content.

The `installed.json` `seed` field records the active strategy. Only one seed should be active at a time — the constraint is documented in every stub SKILL.md and enforced by `npx @extragraj/stackshift-skills repair`.

## Modified files

### `skills/stackshift-core/seeds/_registry.json`
- Added first real entry: `initialvalue-seeding` (tier: optional)
- Updated `description` to document the one-at-a-time invariant and folder naming convention

### `cli/src/prompts.ts`
- Changed `seedCount: number` → `seeds: SeedEntry[]` parameter
- Added `existingSeed?: string` parameter (shows currently active seed as `initialValue`)
- Seed selection step is now **always shown** (previously gated on `seedCount > 0`)
- Selecting "None (skip)" is always a valid option
- Shows a `note` with context about the one-at-a-time constraint and current active seed

### `cli/src/install.ts`
- Added `readExistingSeed()` helper that reads `.stackshift/installed.json → seed`
- Passes `protocolRegistry.seeds` (full array) and `existingSeed` to `runPrompts`

### `cli/src/writer.ts`
- `writeStackshiftMarker` now writes `seed: <id>` to `installed.json` when a seed is selected
- When `seed: 'none'`, the key is **omitted** (backward compatible with existing installs)

### `cli/src/repair.ts`
- Rewritten to handle both protocol bundles and seed folders in a single pass
- New `scanForSeedFolders()` — scans all platforms/scopes for `stackshift-seed-*` folders
- New `removeSeedFromAllLocations()` — removes a seed folder from all platforms/scopes
- New `updateInstalledSeed()` — atomic write of `seed` field in `installed.json`
- New `findSeedEntryByFolder()` / `deriveSeedId()` — maps folder name to registry id
- Five seed cases handled: (0 folders + no recorded), (0 folders + recorded → warn), (1 valid), (1 unknown → warn), (multiple → interactive cleanup prompt)
- Validates recorded seed against registry regardless of folder count
- Combined `outro` reports both protocol and seed repair results

### `cli/src/flags.ts`
- Added `seed?: string` to `Flags` interface
- Added `--seed <id|none>` flag to `parseFlags`
- `validateFlags` reads `flags.seed ?? 'none'` for non-interactive installs
- `showHelp()` documents `--seed` with example

### `skills/stackshift-core/SKILL.md`
- Added "Active seeding strategy" row to lookup router (Section 3)
- Added "Seed Discovery" block documenting the 5-step seed-loading procedure

### `skills/stackshift-core/workflow/2-section-schema.md`
- Added "Check active seed strategy" to Recommended actions — loads seed before writing `initialValue/`

---

## `.stackshift/installed.json` shape (v0.2.0)

```json
{
  "skillVersion": "0.2.0",
  "mode": "recommended",
  "protocols": [...],
  "seed": "initialvalue-seeding"
}
```

When no seed is selected, the `seed` key is omitted entirely (backward compatible).

---

## Breaking changes

None. Existing installs without a `seed` field continue to work without modification.
