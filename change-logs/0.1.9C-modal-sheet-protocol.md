## Protocol: Modal & Sheet

An opt-in system for standalone modal documents that editors create in Sanity Studio and link to from any button or link field via `conditionalLink`. When a link carries `linkType: "linkModal"`, the GROQ query resolves a `modalRef`; `ModalContext.openModalByRef()` intercepts the click and opens a `@stackshift-ui/sheet` or `@stackshift-ui/dialog` overlay instead of navigating.

The protocol separates infrastructure from ongoing work:

- **One-time setup** (`setup.md`) — `conditionalLink` schema changes, modal document schema, desk structure, `ModalContext`, `SmartLink`, GROQ fragment in `pages/api/query.ts`, TypeScript types, preview badge
- **Per-variant workflow** (`variant-template.md`, `integration-with-workflow.md`, `checklist.md`) — structural shell using `@stackshift-ui/sheet` or `@stackshift-ui/dialog`, UI Forge delegation with ModalContext handoff (`openModalByRef`, `closeModal`, `activeModalData`), schema/GROQ registration for each new variant

UI Forge delegation constraint: the protocol provides the overlay shell; UI Forge fills the visual body using the ModalContext shape as named context. No local `useState` for visibility — all open/close/data flows through `useModal()`.
