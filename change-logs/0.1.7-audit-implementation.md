## Fixes

### HIGH

#### H1 ‚Äî `repair.ts` `skillVersion` fallback reads wrong field

**File:** `cli/src/repair.ts`

**Change:** Fallback now reads `existing.skillVersion` instead of `existing.protocols?.[0]?.tier`. Added `skillVersion?: string` to the `InstalledJson` interface. Previous behavior set `skillVersion` to a tier string like `"required"` when the `skill.version` file read failed.

---

### MEDIUM

#### M1 ‚Äî Missing `_registry.schema.json`

**Files:**
- `skills/stackshift-core/_registry.schema.json` (new, shared)
- `skills/stackshift-core/protocols/_registry.json` ‚Äî `$schema` updated to `../_registry.schema.json`
- `skills/stackshift-core/seeds/_registry.json` ‚Äî same

**Change:** A single shared JSON Schema (draft 2020-12) at `skills/stackshift-core/_registry.schema.json` validates both protocol and seed registries. Enforces `id` / `tier` / `title` / `summary` are present, `tier` is one of `required|recommended|optional`, and each entry has exactly one of `file` or `dir`.

#### M2 ‚Äî `ProtocolEntry` interface missing `dir`

**Files:** `cli/src/registry.ts`, `cli/src/writer.ts`, `cli/src/repair.ts`

**Change:** `ProtocolEntry` and `SeedEntry` now declare `file?: string; dir?: string`. `buildCustomProtocolSkill()` renders either path in the SKILL.md table. `writeStackshiftMarker()` emits only the defined field (`file` or `dir`) per protocol entry. `InstalledJson` in `repair.ts` updated to match.

#### M3 ‚Äî `loadSkills()` crashes on missing `skills/` directory

**File:** `cli/src/registry.ts`

**Change:** `readdirSync(skillsDir)` wrapped in try-catch. On failure, throws a descriptive `Error` instead of a raw Node stack trace.

#### M4 ‚Äî Scope prompt labels hardcoded to `.agents/`

**File:** `cli/src/prompts.ts`

**Change:** Labels now read `Project (current directory)` and `Global (home directory)` ‚Äî accurate regardless of platform selection.

#### M5 ‚Äî `installed.json` schema diverges across CLI and bootstrap

**Files:** `skills/stackshift-core/bootstrap/install.md`, `cli/src/writer.ts`

**Change:** Added a canonical schema section to bootstrap Step 7 documenting the full possible shape (including `pendingDesignArchBridge` and `uiForgeIntegration`) with a field-ownership table. `writeStackshiftMarker()` now reads the existing marker and spreads it into the new payload so agent-added fields are preserved across re-installs.

---

### LOW

#### L1 ‚Äî SKILL.md workflow table shows "6 | Verify"

**File:** `skills/stackshift-core/SKILL.md`

**Change:** Removed the row numbered `6` and replaced it with a note below the workflow table clarifying that `workflow/checklist.md` is a post-Step-5 verification pass, not a 6th workflow step.

#### L2 ‚Äî Bootstrap "None" mode is AI-only but not documented

**File:** `skills/stackshift-core/bootstrap/modes.md`

**Change:** Added an `**Availability:**` line to the None section noting it is available only via the AI-agent bootstrap prompt and not via `npx @extragraj/stackshift-skills init`.

#### L3 ‚Äî README Repository Structure missing files

**File:** `README.md`

**Change:** Structure tree now includes `bin/cli.mjs`, `CLAUDE.md`, `_registry.schema.json`, `repair.ts`, and `flags.ts`. Seeds note updated from `v0.1.2` to `v0.1.7`.

#### L4 ‚Äî No post-write validation in install flow

**File:** `cli/src/install.ts`

**Change:** After `writeSelection()` returns, `validateWriteResults()` checks that `stackshift-core` exists on disk for every platform in the results. If any platform is missing, the CLI exits with a clear error message instead of reporting false success.

#### L5 ‚Äî Cross-platform sync is silent

**File:** `cli/src/install.ts`

**Change:** `reportCrossPlatformSync()` emits a `note('Cross-platform sync', ...)` when `writeSelection()` updated platforms the user did not explicitly select. Applied to both interactive and non-interactive flows.

#### L6 ‚Äî Checklist Step 4 missing ui-forge postcondition items

**File:** `skills/stackshift-core/workflow/checklist.md`

**Change:** Added 4 required items under Step 4: preconditions verified, `ui-forge` invoked with `--signal CONVERT_VARIANT --mode body-only`, postconditions passed (FORGE NOTES header, `index.tsx` unchanged, no unexpected files), and no StackShift-managed files modified.

#### L7 ‚Äî `seeds/_registry.json` missing schema

**File:** `skills/stackshift-core/seeds/_registry.json`

**Change:** `$schema` now points to the shared `../_registry.schema.json` created for M1.

#### L8 ‚Äî `loadProtocolRegistry()` can crash on missing file

**File:** `cli/src/registry.ts`

**Change:** `readFileSync` and `JSON.parse` wrapped in try-catch; throws a descriptive error on failure.

#### L9 ‚Äî Registry interface mixes protocols and seeds

**File:** `cli/src/registry.ts`

**Change:** New `loadSeedRegistry()` loads seeds from `seeds/_registry.json` separately. `loadProtocolRegistry()` now only parses the protocols file and delegates seed loading. Interface contract unchanged (still returns `{ protocols, seeds }`), so call sites in `install.ts` were not touched.

---

### OPTIMIZATIONS

#### O1 ‚Äî Atomic JSON writes

**Files:** `cli/src/writer.ts`, `cli/src/repair.ts`

**Change:** Added `writeJsonAtomic()` helper in both files ‚Äî writes to `${filePath}.tmp` then renames. Applied to all lock files and the `.stackshift/installed.json` marker. Prevents corruption if the process is killed mid-write.

#### O2 ‚Äî Interactive install spinner

**File:** `cli/src/install.ts`

**Change:** Added a `spinner()` around `writeSelection()` in the interactive flow, matching the non-interactive flow. Provides progress feedback for large protocol sets or slow filesystems.
