# 0.1.9B — Codex Platform Removal

**Release date:** 2026-04-28
**Previous version:** 0.1.9A
**Version bump:** none — sub-release tracked as 0.1.9B

## Summary

Removes Codex CLI as a separate third installation platform. Codex is now treated as a universal agent that installs to `.agents/skills/` via `npx skills add -a agents`, the same as Amp, Cline, Cursor, DeepSeek, Gemini CLI, Warp, and all other general agents.

**Breaking change for existing scripts:** `--platform codex` now exits with an error. Users who had `--platform codex` in automation scripts should replace it with `--platform agents`.

Existing project installs under `.codex/skills/` continue to work — the UI Forge skill-root fallback lists them as legacy entries so running projects are not disrupted.

---

## CLI Changes

### `--platform codex` removed

**Files:** `cli/src/flags.ts`, `cli/src/prompts.ts`

`Platform` type narrowed from `'agents' | 'claude' | 'codex'` to `'agents' | 'claude'`. Passing `--platform codex` now prints:

```
Invalid --platform value: codex
Valid values: agents, claude, or comma-separated combinations
```

and exits with code 1.

The interactive platform prompt previously offered three options; it now offers two:

- Universal agents (`.agents/`) — recommended
- Claude Code (`.claude/`)

### `writeAgentsMd` removed

**File:** `cli/src/writer.ts`

The function that wrote `AGENTS.md` at install time was exclusively used for Codex CLI skill discovery. It has been deleted along with its call site.

### Platform branching simplified

**Files:** `cli/src/writer.ts`, `cli/src/repair.ts`, `cli/src/install.ts`

All three-way ternaries of the form:

```ts
platform === 'agents' ? '.agents' : platform === 'claude' ? '.claude' : '.codex'
```

reduced to two-way ternaries. Platform detection arrays no longer include `'codex'`. `install.ts` removes the codex lock-file read and the `agentsMdNote` from both output paths.

## Migration

Users with `--platform codex` in install scripts:

```bash
# Before (errors in 0.1.9B)
npx @extragraj/stackshift-skills init --platform codex --no-interactive

# After
npx @extragraj/stackshift-skills init --platform agents --no-interactive
```

Users with skills under `.codex/skills/` from a prior install do not need to reinstall — the skill continues to load from that path. To migrate to `.agents/skills/` run the new install command above.
